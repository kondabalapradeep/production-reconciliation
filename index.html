Method & rules baked into this report
  • Finished goods: Sage production entries — JR (work ticket receipts) + BR (BOM production receipts) — per item for the selected day or full week. Sage Cost = quantity × unit cost (BOM roll-up: sum of all component quantities × their CI_Item LastTotalUnitCost, through every BOM level).
  • Raw materials: Theoretical = Sage production × BOM QuantityPerBill (material lines only, no scrap %). Because bulk/batch production is itself recorded in Sage, each production level generates demand for its own components. Actual = JI (work ticket issues) + BI (backflush issues). Deviation = Actual − Theoretical; Deviation Cost = deviation × LastTotalUnitCost. Actual above theoretical includes real-world scrap and waste.
  • Day filter: toggles restrict both production and consumption to that transaction date. Note: material issues sometimes post a day apart from the production receipt of the same ticket, so single-day deviations can be noisier than the full-week view.
  • Categories are inferred from item code prefix: A = Aerosol, P = Pops, H = Rainbow/Drinks, W = Water 1 Gal, L = Water 2.5 Gal; everything else = Batch/Other. Finished goods list in that order, with batches always at the bottom.
  • Used In shows the finished goods a component flows into for this period's production. When a component belongs to a batch, the batch is skipped and the finished goods that consume the batch are shown instead.
  • Nothing uploads to any server — files are read locally in the browser.
